Apercu rapide

MirzAI est un toolkit pour piloter un Mac Mini Apple Silicon comme serveur d'IA headless depuis n'importe quel poste Linux. Le Mac tourne les modeles en local via MLX. Vous le controlez via le CLI mirza ou cette interface web.

Onglets de l'interface

OngletDescription
DashboardStatut du serveur, infos hardware (puce, RAM, GPU), modele actif, actions rapides (reveil, veille, reboot, stop MLX), et logs en direct du moteur d'inference.
ChatChat avec streaming, multi-provider (MLX local, OpenAI, Anthropic, Groq, Mistral, Ollama), historique des conversations stocke dans le navigateur, et support MCP.
ModelesCatalogue en temps reel depuis l'API HuggingFace (organisation mlx-community). Chaque carte montre la taille estimee, le KV Cache, les tok/s prevus pour votre puce, et bloque les modeles trop gros pour votre RAM.
ConfigAffichage de mirza.conf. Le bouton "Regenerer" relance un scan SSH complet du Mac.
MonitoringDashboard Grafana integre en mode kiosk (sans menus). Affiche CPU, GPU, ANE, temperature, et memoire en temps reel.
DocumentationCe que vous lisez actuellement.

Commandes CLI essentielles

CommandeAction

mirza startReveiller le Mac via Wake-on-LAN.
mirza sshSession SSH interactive sur le Mac.
mirza modelsCatalogue de modeles compatibles avec votre config.
mirza deploy <id>Telecharger un modele sur le Mac.
mirza serve [id]Demarrer le serveur d'inference MLX.
mirza stop-mlxArreter le serveur d'inference.
mirza uiLancer cette interface web (port 3333).
mirza stop-uiArreter la WebUI.
mirza config --refreshRegenerer la configuration en interrogeant le Mac.

Comprendre les cartes de modeles

Chaque carte de modele dans l'onglet Modeles affiche des metriques calculees dynamiquement :

  • Poids : taille du modele en Go, calculee comme Parametres * (Bits / 8).
  • KV Cache : memoire estimee pour le cache d'attention (1.5 Go pour les modeles legers, 3 Go pour les modeles >15B parametres).
  • RAM OS : 6 Go reserves systematiquement pour macOS et les services de fond.
  • tok/s : vitesse de decodage estimee, calculee selon la bande passante memoire de votre puce (ex: M4 = 120 Go/s, M4 Max = 546 Go/s).
  • Flash Attention / Paged Attention : optimisations supportees nativement par le framework MLX sur Apple Silicon.

Si un modele depasse la RAM disponible (poids + KV Cache + 6 Go OS > RAM), la carte est grisee et les boutons sont desactives. On ne vous laissera pas crash votre serveur par optimisme.

Depannage rapide

ProblemeSolution
mirza: command not foundLancez source ~/.bashrc ou relancez votre terminal.
Dashboard videLe Mac est probablement eteint ou le SSH echoue. Verifiez avec mirza status.
Port 3333 deja utiliseLancez mirza stop-ui puis mirza ui.
Grafana ne charge pasVerifiez que le service tourne : mirza ssh puis brew services list.
Modeles trop lentsVerifiez la pression memoire dans Grafana. Si le Mac swappe, le modele est trop gros.
